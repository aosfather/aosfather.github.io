# QuantEye —— 你的「本地化」私人盯盘哨兵

QuantEye 是一款完全运行在你电脑上的桌面端盯盘软件。​

我们不提供网页版，也不把你的策略存在云端。因为我们知道，对于交易来说，速度、隐私和稳定比花哨的界面更重要。

## 一、为什么必须是「本地软件」？

针对上班族和冲动型交易者，云端软件有两个致命伤：卡顿​ 和 泄密。

QuantEye 的本地化架构，正是为了根治这两点。

### 1. 彻底断绝“云端杀熟”与隐私焦虑

场景
云端软件风险
QuantEye 本地方案
策略存储​
你的策略逻辑上传到服务器，存在被平台方或黑客窥探的风险。
策略只存在你的电脑里。​ 哪怕断网，软件照样跑。
账号安全​
浏览器登录，Cookie 容易被劫持，甚至误点钓鱼网站。
本地客户端直连。​ 无需浏览器，减少攻击面。
数据主权​
历史数据在别人服务器上，说删就删。
SQLite 本地数据库。​ 所有行情快照、报警记录永久归你所有。
### 2. 零延迟：不跟服务器“挤独木桥”

上班族时间碎片化，每一秒都珍贵。

QuantEye 采用 WebSocket 直连数据源 + 本地内存计算：
没有 HTTP 请求排队：不像网页刷新需要等待服务器响应。
多线程并行监控：同时盯 500 只股票/币种，CPU 分配给你自己说了算，不会因为平台限流而漏报。
### 3. “老板键”与工作模式

作为一款给上班族设计的软件，隐蔽性至关重要。
一键隐身（Boss Key）：按下预设快捷键（如 Ctrl+~），软件瞬间隐藏，连任务栏图标都能消失。
无感运行：最小化到系统托盘，占用极低内存，不弹窗打扰你正常工作（除非触发警报）。
## 二、它是如何解决你痛点的？

痛点 A：上班没空看，回家拍断腿

解决方案：后台硬核值守

QuantEye 作为一个桌面进程，即使你把窗口关了，它依然在后台像个老管家一样守着：
开机自启：早上上班开机，它自动开始干活，不需要你手动点开。
本地弹窗 + 语音播报：不需要装任何插件，直接调用 Windows 系统通知。哪怕你在敲代码，也能听到“贵州茅台，突破1700”的语音提示。
离线缓存：如果公司网络波动断了，软件会缓存数据，恢复后自动补齐，不会漏掉关键K线。
痛点 B：脑子一热就冲，纪律崩塌

解决方案：把“纪律”锁死在本地代码里

既然是本地软件，我们可以做得更“硬”：
策略脚本化（No-Code/Low-Code）：
你可以在本地编辑一个简单的 .txt或 .json规则文件：
json
{
  "name": "均线回踩策略",
  "condition": "CLOSE > MA20 AND VOL < REF(VOL,1)*0.8",
  "action": "ALERT_AND_SNAPSHOT"
}
软件只认这个文件，你没改它，它就永远按这个执行，不会因为心情不好而改变策略。
硬件级风控：
配合本地防火墙规则，甚至可以设置“交易时间段内禁止访问交易所网页”，物理隔绝手痒。
### 三、软件界面与交互（桌面端体验）

QuantEye 的 UI 设计遵循 “暗色系 · 极客风 · 低干扰”：
多屏适配：如果你是职业玩家，支持拖拽多个窗口到副屏（比如左边看盘口，右边看预警日志）。
自定义布局：所有面板（行情列表、K线、资金流）都是悬浮窗，想怎么摆就怎么摆。
本地数据回溯：
点击历史警报，软件直接从你电脑硬盘里调取当时的盘口数据回放，不需要重新下载，秒开。
四、技术栈简述（给懂行的人看）
语言/框架：Python (PySide6/PyQt) 或 C++ (Qt)，主打轻量级与跨平台潜力（目前主攻 Windows）。
数据存储：LevelDB 或 SQLite，用于高速读写 Tick 数据和报警日志。
通信机制：原生 Socket / WebSocket 连接行情源，不经过中间商转发。
资源占用：空闲时 CPU < 1%，内存占用约 100MB-300MB（取决于监控数量）。
